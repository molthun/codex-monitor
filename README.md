# CodexMonitor

Rainmeter desktop widget for Windows hardware monitoring.

The widget is intentionally configured as part of the desktop, not as an always-on-top overlay. Hardware sensors come from a bundled CodexBridge executable using LibreHardwareMonitor, and Rainmeter reads a flat text file generated by that bridge.

## Current Status

- Production install root: `C:\CodexMonitor`
- Active Rainmeter skin: `<RainmeterSkinPath>\CodexMonitor\CodexMonitor.ini`
- Sensor output file: `C:\CodexMonitor\@Resources\temps.txt`
- Bridge scheduled task: `CodexMonitor Bridge Elevated`
- Display watcher: `C:\CodexMonitor\Watch-PrimaryDisplay.ps1`
- Optional local staging/reinstall kit: `<LocalStagingFolder>`
- Planned source of truth for collaboration: GitHub repository

## Main Features

- CPU load, CPU temperature, CPU fan RPM.
- RAM usage.
- GPU load, GPU temperature, GPU fan RPM and percent.
- Board/case/PSU fan display based on hardware sensor mapping.
- Ethernet, Wi-Fi, and Wi-Fi AP traffic split.
- Network graph and download/upload bars.
- Per-disk I/O for C:, D:, E:.
- Disk used/free bars for C:, D:, E:.
- Automatic 1080p/4K skin profile selection.
- Automatic positioning on the current primary monitor.

## Required Software

- Windows 11.
- Rainmeter 4.5.x or newer.
- NVIDIA driver with `nvidia-smi.exe` available.
- The widget is display-only. Fan behavior is left to the user's BIOS/UEFI, drivers, or existing system tools.

## Configuration

The project is configured through:

```text
config.json
```

`config.json` is the local working config and is ignored by Git. On first install, it is created from the public template:

```text
config.example.json
```

Edit `config.json` directly on your workstation. Scripts pick it up on the next run; long-running services should be restarted after config changes.

To install and configure everything from scratch on a clean PC, run this one-liner in an elevated PowerShell console:

```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force; irm https://raw.githubusercontent.com/molthun/codex-monitor/main/Deploy/Bootstrap-CodexMonitor.ps1 | iex
```

This bootstrap command will:
1. Request **Administrator** elevation.
2. Download the latest public source ZIP from GitHub to `C:\CodexMonitor`.
3. Save a local version marker so future background updates can compare against GitHub.
4. Hand off execution to the main installer (`Setup-CodexMonitor.ps1`), which will:
   - Detect and install all remaining prerequisites (**Rainmeter**) using `winget`.
   - Run the interactive **Configuration Wizard** (`Configure-CodexMonitor.ps1`) to choose widget scaling (Auto, 1080p, 4K), choose which local disk drives to display, and set up network interface exclusions.
   - Copy the bundled bridge executable, configure the startup shortcut, register the elevated scheduled tasks, configure active `Rainmeter.ini` layout/coordinates, and load/activate the CodexMonitor skin.

## Source Of Truth

Going forward, GitHub should be the primary source of truth for developers:

- source code;

- project documentation;
- changelog.

Any local reinstall/staging folder is only a temporary recovery copy until the GitHub repo and release workflow are created. Do not treat a local zip archive as the canonical project state after GitHub is live.

## Daily Commands

Refresh the skin:

```powershell
& "C:\Program Files\Rainmeter\Rainmeter.exe" !Refresh "CodexMonitor"
```

Switch profile automatically:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Switch-WidgetSize.ps1" -Mode Auto
```

Force 1080p:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Switch-WidgetSize.ps1" -Mode 1080p
```

Force 4K:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Switch-WidgetSize.ps1" -Mode 4K
```

Restart widget services:

```powershell
Stop-ScheduledTask -TaskName "CodexMonitor Bridge Elevated"
Start-ScheduledTask -TaskName "CodexMonitor Bridge Elevated"
& "C:\Program Files\Rainmeter\Rainmeter.exe" !Refresh "CodexMonitor"
```

## Upgrading Prerequisites

The background display watcher checks for available Rainmeter updates once every 24 hours. If an update is found, a Windows GUI prompt will appear. Clicking **Yes** will automatically perform the update safely (closing applications to prevent file locks, running `winget` silently, and restarting everything cleanly).

You can also trigger this update manually at any time by running:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Upgrade-Prerequisites-And-Apps.ps1"
```

## Current Size Profiles

- 1080p: about `430 x 950`.
- 4K: about `720 x 1500`.

The 4K profile is intentionally larger, but still uses a single-column layout so all values remain visible at a glance.
