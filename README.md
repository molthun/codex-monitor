# CodexMonitor

Rainmeter desktop widget for Windows hardware monitoring.

The widget is intentionally configured as part of the desktop, not as an always-on-top overlay. Hardware sensors come from FanControl through a .NET 10 bridge, and Rainmeter reads a flat text file generated by that bridge.

## Current Status

- Production install root: `C:\CodexMonitor`
- Active Rainmeter skin: `<RainmeterSkinPath>\CodexMonitor\CodexMonitor.ini`
- Sensor output file: `C:\CodexMonitor\@Resources\temps.txt`
- Bridge scheduled task: `CodexMonitor Bridge Elevated`
- Display watcher: `C:\CodexMonitor\Watch-PrimaryDisplay.ps1`
- Optional local staging/reinstall kit: `<LocalStagingFolder>`
- Planned source of truth for collaboration: GitHub repository

## Main Features

- CPU load, CPU temperature, CPU fan RPM.
- RAM usage.
- GPU load, GPU temperature, GPU fan RPM and percent.
- Board/case/PSU fan display based on FanControl sensor mapping.
- Ethernet, Wi-Fi, and Wi-Fi AP traffic split.
- Network graph and download/upload bars.
- Per-disk I/O for C:, D:, E:.
- Disk used/free bars for C:, D:, E:.
- Automatic 1080p/4K skin profile selection.
- Automatic positioning on the current primary monitor.

## Required Software

- Windows 11.
- Rainmeter 4.5.x or newer.
- FanControl v268 or newer.
- .NET 10 SDK when installing directly from source, because `CodexBridge.exe` is built locally.
- NVIDIA driver with `nvidia-smi.exe` available.

## Configuration

The project is configured through:

```text
config.json
```

`config.json` is the local working config and is ignored by Git. On first install, it is created from the public template:

```text
config.example.json
```

Edit `config.json` directly on your workstation. Scripts pick it up on the next run; long-running services should be restarted after config changes.

After cloning from GitHub, the normal flow is:

```powershell
cd C:\CodexMonitor
powershell -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Install-Prerequisites-And-Restore.ps1"
powershell -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Install-CodexMonitor.ps1"
```

## Source Of Truth

Going forward, GitHub should be the primary source of truth for developers:

- source code;
- Rainmeter skin and presets;
- install/watch/switch scripts;
- project documentation;
- changelog.

Any local reinstall/staging folder is only a temporary recovery copy until the GitHub repo and release workflow are created. Do not treat a local zip archive as the canonical project state after GitHub is live.

## Daily Commands

Refresh the skin:

```powershell
& "C:\Program Files\Rainmeter\Rainmeter.exe" !Refresh "CodexMonitor"
```

Switch profile automatically:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Switch-WidgetSize.ps1" -Mode Auto
```

Force 1080p:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Switch-WidgetSize.ps1" -Mode 1080p
```

Force 4K:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File "C:\CodexMonitor\Deploy\Switch-WidgetSize.ps1" -Mode 4K
```

Restart widget services:

```powershell
Stop-ScheduledTask -TaskName "CodexMonitor Bridge Elevated"
Start-ScheduledTask -TaskName "CodexMonitor Bridge Elevated"
& "C:\Program Files\Rainmeter\Rainmeter.exe" !Refresh "CodexMonitor"
```

## Current Size Profiles

- 1080p: about `430 x 950`.
- 4K: about `720 x 1400`.

The 4K profile is intentionally larger, but still uses a single-column layout so all values remain visible at a glance.
